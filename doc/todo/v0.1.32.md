## 2. 🔌 Passe drivers (asymétries réelles)

> La plupart des « trous » signalés par l'audit étaient déjà comblés. Voici ce qui
> reste réellement à faire après vérification.

- [x] ✅ **Maintenance sur Neon / Supabase / TimescaleDB** — *fait (helper `pg_compat::maintenance_operations` + `pg_compat::run_maintenance`, délégué par les 4 drivers PG ; cargo check OK)*.
  Le trait par défaut renvoie `supports_maintenance()=false` +
  `run_maintenance()=NotSupported` (`traits.rs:584-600`). Seul `postgres.rs:496+`
  l'override. Ces trois wrappers PG héritent donc du défaut → pas de VACUUM /
  ANALYZE / REINDEX alors que la base le permet.
  - **Action** : extraire la logique de `postgres.rs` (run/list maintenance) vers un
    helper `pg_compat::*` réutilisable, puis l'exposer dans `neon.rs`, `supabase.rs`,
    `timescaledb.rs` (override `supports_maintenance` + délégation).
  - *Vérif* : sur une connexion Neon, le menu maintenance liste VACUUM/ANALYZE/REINDEX
    et `run_maintenance` s'exécute sans `NotSupported`.
  - *Effort* : faible (~15-20 lignes/driver après extraction du helper).
  - *Note* : TimescaleDB peut justifier des ops spécifiques (compression de chunks) —
    hors scope ici, basique d'abord.
- [ ] ⏸️ **MongoDB — EXPLAIN** — *différé après audit*. EXPLAIN n'est **pas** une
  mécanique driver : `QueryPanel.tsx:785-799` construit une requête `EXPLAIN`
  spécifique au dialecte SQL et `explainPlanParser.ts` ne parse que les formats JSON
  **PostgreSQL/MySQL**. L'`explain` Mongo renvoie une structure radicalement différente
  (`queryPlanner.winningPlan`, stages `COLLSCAN`/`IXSCAN`) → nécessite construction de
  requête Mongo + **nouvelle branche de parser** + mapping UI. Item moyen-gros, pas un
  quick win → chantier dédié hors de cette release.
- [ ] 🔍 **Redis — streaming** : confirmer que `execute_stream` n'est pas implémenté,
  puis l'ajouter via boucle `SCAN` → `StreamEvent` (cohérence avec les autres drivers
  qui exposent `supports_streaming=true`).
  - *Vérif* : navigation d'un grand keyspace Redis sans charger tout en mémoire.
  - *Effort* : moyen. **Optionnel** — ne garder que si le temps le permet.
