# 🚀 **V2 — “L'outil indispensable du Power User”**

> **Objectif** : Dépasser les outils classiques en offrant une expérience de manipulation de données plus rapide, plus sûre et plus visuelle.

### ⚡ Smart Grid & Édition (Productivité accrue)

- [x] **Édition hybride intelligente** :
- Clic/Double-clic sur une cellule pour édition SQL inline directe.
- Zone spécifique (ex: icône "expand") pour ouvrir la modale d'édition complète.
- La selection d'une ligne ne doit plus se faire via un clic sur la ligne mais via un clic sur la colonne checkbox.

- [x] **Peek Definition (Aperçu de relation)** : Au survol d'une clé étrangère, affichage d'une "popover" avec les données de la ligne liée sans changer de table. + Pouvoir cliquer sur la relation pour ouvrir la table liée.
- [x] **Mode “Bac à sable” (Sandbox)** :
- Modification locale des données sans impact immédiat.
- Génération automatique du script SQL de migration correspondant aux changements effectués manuellement.

### 🎨 Visualisation & Architecture

- [x] **ER Diagram Interactif** : Génération automatique d'un schéma relationnel beau et fluide (type "Canvas").
- Navigation facilitée : cliquer sur une table dans le schéma l'ouvre dans l'explorateur.

- [x] **Visual Data Diff** : Comparaison côte à côte (type "Git diff") du contenu de deux tables ou de deux résultats de requêtes (ex: comparer Prod vs Staging).
- [x] **Indicateurs de structure** : Visualisation claire des index et des contraintes directement dans le flux de données.

### 🛠️ Gestion BDD avancée (Combler le retard sur phpMyAdmin)

- [x] **Opérations de maintenance** : Optimisation des tables, réparation, et changement de moteur de stockage.
- [ ] **Gestionnaire de Routines & Procédures** : Interface dédiée pour créer et éditer les fonctions et procédures stockées.
- [ ] **Triggers & Événements** : Vue liste et édition des déclencheurs et des tâches planifiées (Events) en base.
- [x] **Recherche globale (Full-text)** : Pouvoir chercher une chaîne de caractères dans toute la base de données (toutes les tables/colonnes).

### 📤 Collaboration & Partage "Local-First"

- [x] **Export "Self-contained HTML"** : Générer un fichier HTML unique contenant les données et une interface de filtrage/tri minimale pour partage rapide.
- [ ] **Snapshots de résultats** : Sauvegarder un état précis d'un résultat de requête pour comparaison ultérieure.
- [ ] **Génération de liens de partage** : Exportation sécurisée vers un service de partage temporaire (optionnel).

### 🏗️ Core Engineering & Backend (Performance & Stabilité)

- [x] **Streaming Data Pipeline** : Architecture de flux pour exports/résultats massifs (0% OOM).
- [x] **Virtual Relations Engine** : Définir des relations (FK) virtuelles dans QoreDB (ex: Join entre 2 collections Mongo ou table SQL mal conçue).
- [x] **Background Job Manager** : Gestion asynchrone pour les tâches longues (Exports, Migrations, Gros updates) avec reprise sur erreur.
- [x] **Universal Query Interceptor** : Hooks pour profiling, logging d'audit, et prévention de requêtes destructrices (Safety Net backend).

### 🧱 Extension du Data Engine

- [ ] **Nouveaux Drivers** :
  - [x] **SQLite** (Support 1st class : WAL mode, extensions).
  - [x] **Redis** (Mode key-value + visualisation JSON si applicable).
  - [x] **DuckDB** (Pour l'analyse de fichiers locaux CSV/Parquet - _Exploration_).

- [ ] **Connection Resilience** :
  - [ ] **Tunneling SSH Persistant** : Reconnexion automatique, support des clés Ed25519/Agents.
  - [ ] **Smart Keep-Alive** : Détection proactive des timeouts.
